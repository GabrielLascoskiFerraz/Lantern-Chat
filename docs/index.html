Everything your LAN needs

Focused communication, without the cloud-shaped overhead.

Lantern combines a native desktop experience with a single relay service that coordinates presence, delivery, and shared group state.

↔

Direct and group chat

Real-time one-to-one messaging plus groups with names, emoji, colors, descriptions, participants, owners, and administrators.

◎

Rich message controls

Reply, react, forward, favorite, pin, search, edit recent sent messages, delete for yourself or everyone, and archive conversations.

⇩

Files and images

Share attachments and large files. Group attachments use temporary relay caching for delivery and redownload.

!

Network-wide announcements

Publish announcements, review reactions and read confirmations, with relay-side expiration after 24 hours.

◉

Presence and notifications

Online status, unread indicators, native notifications, tray behavior, and optional launch at startup.

GIF

Relay-hosted stickers

Serve validated GIF sticker collections from the relay, organized by folders and refreshed without restarting it.

☾

Do Not Disturb

Silence notifications for 15 minutes, one hour, or until tomorrow.

⇧

Backup and export

Back up and restore local data, and export conversations as TXT or HTML.

▦

Relay dashboard

Inspect connected clients, uptime, and announcements. Add a token outside a trusted LAN.

How it works

One relay. Every conversation.

  1. 01

    Choose the relay machine

    Run one LanternRelay instance on a computer that stays reachable on the LAN. The default port is 43190.

  2. 02

    Open Lantern on each desktop

    Each device keeps its own local profile and SQLite database. No account registration is required.

  3. 03

    Discover automatically

    Clients find the relay through mDNS, or you can enter the relay host and port manually in Settings.

  4. 04

    Chat across the LAN

    The relay coordinates presence, messages, groups, announcements, reactions, synchronization, and attachment delivery.

Security & privacy

Local-first by architecture.

Lantern is intended for trusted local networks. Profiles and conversation history are stored locally on client devices, while communication passes through the relay you operate.

No user accountsIdentity is a local profile tied to each device.
No third-party sticker providerGIF catalogs are served by your relay.
Configurable dashboard accessAn optional token protects the dashboard on untrusted networks.
Clear deployment boundaryOpen the relay port only where required and keep mDNS within the intended network.

Important: The current codebase does not claim end-to-end encryption. Use Lantern only on networks whose relay and infrastructure you trust.

Installation

Download a release or build from source.

Install the client on each workstation and run exactly one relay instance on the chosen server computer.

Open ReleasesFull documentation →
Build from source
# Node.js 22 LTS and npm 10+
git clone https://github.com/GabrielLascoskiFerraz/Lantern-Chat.git
cd Lantern-Chat
npm install

# Client + local relay
npm run dev

# Build for current OS
npm run build

Network setup: Allow TCP port 43190 on the relay machine. Permit mDNS for discovery, or configure the relay IP and port manually.
